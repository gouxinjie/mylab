# 后端专用 AGENTS.md 模板

下面是一份面向后端服务的 AGENTS.md 模板，强调接口规范、数据库安全与错误处理。

## 模板内容

```markdown
# 后端开发规范（Node.js / Python）

## 一、代码规范
- 使用 TypeScript（Node）或类型注解（Python），禁止裸类型
- 遵循统一 Lint 规则，函数必须有注释
- 禁止未使用的变量、函数与 import

## 二、接口规范
- 统一响应结构：{ success, code, message, data }
- 所有请求必须校验参数
- 错误必须捕获并返回中文错误描述
- 不允许编造接口，不明确必须询问

## 三、数据库规范
- 必须使用参数化查询，禁止拼接 SQL
- 表结构与字段必须有中文注释
- 不允许假设数据库结构

## 四、安全规范
- 敏感信息使用环境变量，禁止硬编码
- 所有写操作必须鉴权，禁止跳过
- 用户输入必须校验与转义

## 五、日志规范
- 错误必须记录日志
- 禁止输出敏感信息（token、密码）
- 生产环境移除调试日志
```

## 统一响应示例

```json
{
  "success": true,
  "code": 200,
  "message": "操作成功",
  "data": {}
}
```

## 使用建议

- 与前端约定同一套响应结构，减少联调成本
- 把鉴权、错误码等团队规范固化进模板

> 后端规范的核心：**安全、可预测、可追溯。**
