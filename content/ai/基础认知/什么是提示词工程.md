# 什么是提示词工程

## 定义

**提示词工程（Prompt Engineering）**是通过精心设计输入给大语言模型的指令（Prompt），来引导模型输出更准确、更符合预期结果的一门实践技巧。

## 一个好提示词的要素

- **角色（Role）**：告诉模型它是谁，如"你是一位资深前端工程师"
- **任务（Task）**：明确要做什么
- **上下文（Context）**：提供必要的背景信息
- **约束（Constraints）**：格式、字数、语气等限制
- **示例（Examples）**：给出输入输出样例（Few-shot）

## 常用技巧

### 1. 零样本与少样本

- **Zero-shot**：直接提问，不给示例
- **Few-shot**：给出几个示例，让模型模仿

### 2. 思维链（Chain of Thought）

引导模型"一步一步思考"，显著提升推理类任务的准确率：

```text
请一步一步分析这道题，先列出思路，再给出最终答案。
```

### 3. 结构化输出

要求模型以固定格式（如 JSON）返回，便于程序解析：

```json
{
  "summary": "文章摘要",
  "tags": ["标签1", "标签2"]
}
```

## 一个对比示例

| 写法 | 效果 |
| --- | --- |
| "写个函数" | 模糊，输出不可控 |
| "用 TypeScript 写一个防抖函数，带注释和类型定义" | 精确，可直接使用 |

> 提示词工程的核心：**把模糊的需求，翻译成清晰、可执行的指令。**
