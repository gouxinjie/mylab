# 什么是 RAG 检索增强生成

## 定义

**RAG（Retrieval-Augmented Generation，检索增强生成）**是一种让大语言模型在回答问题前，先从外部知识库中"检索"相关资料，再基于这些资料"生成"答案的技术方案。

## 为什么需要 RAG

大语言模型存在两个天然短板：

1. **知识截止**：训练数据有时间上限，无法知道最新信息
2. **幻觉（Hallucination）**：可能一本正经地编造不存在的内容

RAG 通过引入可信的外部资料，有效缓解上述问题，尤其适合**企业知识库、文档问答**等场景。

## RAG 工作流程

```text
1. 文档切分（Chunking）      把资料切成小块
2. 向量化（Embedding）        把每块转换为向量
3. 存入向量数据库（Vector DB）
4. 用户提问 → 问题向量化 → 检索最相似的若干块
5. 把检索结果拼进 Prompt → 交给 LLM 生成最终答案
```

## 核心组件

- **Embedding 模型**：负责把文本转换为向量
- **向量数据库**：如 Milvus、Pinecone、pgvector，负责相似度检索
- **重排（Rerank）**：对检索结果二次排序，提升相关性

## 简单示意

```text
用户："我们公司的年假政策是什么？"
  ↓ 检索企业制度文档
  ↓ 找到《员工手册·第 3 章》
  ↓ 拼入 Prompt
LLM："根据《员工手册》，入职满一年可享 5 天年假……"
```

> RAG 的价值：**让模型"开卷考试"，答案有据可查。**
