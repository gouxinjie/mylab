# 封面图不用裁切，也能铺满又好看：同图模糊背景法

> 关键词：封面图、object-fit、contain、模糊背景、Next.js、CSS

## 一、背景与问题

在前端项目（尤其是作品集 / 项目展示类页面）里，我们常遇到这样的尴尬：

- 封面图来源五花八门：有的是手机截图（竖向 9:16），有的是网页截图（横向 16:9），有的是方形的 Logo 图。
- 如果统一用 `object-fit: cover` 把图片**裁切**铺满固定比例的容器，竖向图会被放大到"顶天立地"，横向图只裁一点点，观感极不一致；而且核心内容可能被裁掉。
- 如果改用 `object-fit: contain` 让图片**完整显示**，比例不符的地方就会留出容器底色（常见的浅灰 / 白），看起来很"露怯"，不够精致。

矛盾点在于：

> 既要「整图完整可见、不被裁切」，又要「容器被铺满、不露出底色」。

## 二、核心思路：用图片自己当自己的调色板

最优雅的解法，是**为容器加一层"同一张图的放大模糊背景"**：

1. 背景层：同一张图，`object-fit: cover` 铺满整个容器，再做高斯模糊 + 轻微放大。
2. 前景层：同一张图，`object-fit: contain` 完整居中显示。

因为背景就是前景本身，模糊后留白区显示的颜色，正是**这张图主体色调的展开**。底和图是同一套配色，自然融合——类似音乐 App / 影视详情页那种"模糊大图 + 清晰封面"的磨砂质感。

关键点：**这个方案不需要任何取色算法**，零额外依赖，而且 Next.js 对同一 `src` 的图片会复用缓存，模糊底图几乎不增加网络开销。

## 三、实现步骤

下面以 Next.js + SCSS Modules 为例（本项目 `components/business/FeaturedProjects`）。

### 1. 容器固定高度（保证同高、不随比例变形）

```scss
.card__image-box {
  position: relative;
  width: 100%;
  height: 220px;        // 固定高度，所有封面同高
  overflow: hidden;
  border-radius: var(--radius-lg) var(--radius-lg) 0 0;
  background-color: var(--color-surface-muted);
}
```

### 2. 两层图片

```tsx
<div className={styles['card__image-box']}>
  {/* 模糊背景层：同图放大模糊铺满容器 */}
  <Image
    src={project.covers[0]}
    alt=""
    aria-hidden
    fill
    quality={80}
    sizes="(min-width: 1024px) 384px, (min-width: 640px) 50vw, 100vw"
    className={styles['card__cover-bg']}
    loading="lazy"
  />
  {/* 清晰前景层：完整显示，不裁切 */}
  <Image
    src={project.covers[0]}
    alt={`${project.title} 封面`}
    fill
    quality={95}
    sizes="(min-width: 1024px) 384px, (min-width: 640px) 50vw, 100vw"
    className={styles.card__cover}
    loading="lazy"
  />
</div>
```

### 3. 两层样式

```scss
// 模糊背景层
.card__cover-bg {
  width: 100%;
  height: 100%;
  object-fit: cover;
  filter: blur(24px);   // 高斯模糊，抹掉细节只留大色块
  transform: scale(1.2); // 放大溢出，避免模糊边缘露出底色
  opacity: 0.9;
}

// 清晰前景层
.card__cover {
  width: 100%;
  height: 100%;
  object-fit: contain;  // 完整显示，不裁切
  object-position: center;
}
```

> DOM 顺序保证：背景层 → 前景层 → 其它交互元素（如轮播箭头）。无需 `z-index` 即可让按钮绘制在最上层。

## 四、原理拆解（为什么有效）

| 手段 | 作用 |
| --- | --- |
| 背景 = 前景同图 | 留白区显示的是"图自身色调的模糊展开"，底图配色天然和谐，不存在生硬底色 |
| `object-fit: cover` | 让背景图铺满整个容器，不漏边 |
| `filter: blur(Npx)` | 高斯模糊抹掉细节，只保留大块色彩，形成磨砂质感 |
| `transform: scale(1.2)` | 模糊核会向边缘溢出产生半透明渐隐，放大 1.2 倍把渐隐区推到容器外，四边严丝合缝 |
| `opacity: 0.9` | 让背景略微透明，叠加容器底色后更柔和（可调 0.7~1 控制浓度） |
| `quality={80}`（背景） | 背景图无需高清，降低体积；前景 `quality={95}` 保证清晰 |

## 五、配套：让卡片本身尺寸一致

封面容器宽度会跟随网格列宽。如果网格用 `1fr`（等价于 `minmax(auto, 1fr)`），某张卡片最小内容偏宽时会把整列撑大，导致列不等宽、卡片大小偏差。修法：

```scss
.grid {
  display: grid;
  gap: 16px;

  @media (min-width: 640px) {
    grid-template-columns: repeat(2, minmax(0, 1fr)); // 强制等宽
  }
  @media (min-width: 1024px) {
    grid-template-columns: repeat(3, minmax(0, 1fr));
  }
}

.card {
  min-width: 0; // 防止被内部内容撑开
}
```

## 六、方案的边界与升级

### 当前方案的短板

如果遇到**白底 / 纯色底图**（例如一张居中 Logo 在纯白背景上），模糊后整个底仍是白的，留白区依旧发白。此时才需要真正的"主题色提取"。

### 升级方向 A：服务端取主色（推荐）

用 `sharp` 把图缩到极小尺寸（如 1×N 像素），取平均 / 主导色，写入数据字段 `themeColor`，再用 `background-color: var(--theme)` 铺底，上叠一层半透明模糊图。一次计算、长期复用。

### 升级方向 B：前端 Canvas 取色（运行时）

把图画到 1px 的 Canvas，`getImageData` 取主色并 set 到 CSS 变量。成本略高，且需处理跨域。

## 七、小结

- 想要「整图完整 + 容器铺满 + 不露底色」，**同图模糊背景法**是最省事、零依赖、效果又好的方案。
- 核心一句话：**让图片自己模糊后当自己的背景，cover 铺底、contain 前景。**
- 截图类封面（app / 网页截图）色彩丰富，此法观感最佳；白底 Logo 图才需要额外取色兜底。

## 八、环境依赖与可移植性说明（复现前必读）

本文示例基于本项目的真实代码，其中有几处**项目专属约定**，外部读者复现时需注意替换：

1. **CSS 变量是项目 Design Token**，并非浏览器内置。文档中用到的：
   - `var(--color-surface-muted)`：容器占位底色（浅灰），可替换为 `#f3f4f6` 之类的具体色值
   - `var(--radius-lg)`：大圆角，可替换为 `16px`
   - `var(--color-accent)`：主题色（hover 边框），可替换为任意品牌色
   - `var(--color-text-primary)`：正文色，仅用于 `color-mix` 生成分隔线，可替换为 `#111827`
   - `var(--color-bg)`：区块背景色

   若直接复制代码报错，把这些 `var(--xxx)` 换成具体色值即可。

2. **`next/image` 的特殊要求**：示例用 `<Image fill />`，它要求**父容器必须 `position: relative`**（本文 `.card__image-box` / `.carousel` 已满足）。若你用原生 `<img>`，写法改为：

   ```html
   <div class="image-box">
     <img class="cover-bg" src="cover.jpg" alt="" aria-hidden="true" />
     <img class="cover" src="cover.jpg" alt="封面" />
   </div>
   ```

   ```css
   .image-box { position: relative; width: 100%; height: 220px; overflow: hidden; }
   .cover-bg, .cover { position: absolute; inset: 0; width: 100%; height: 100%; }
   .cover-bg { object-fit: cover; filter: blur(24px); transform: scale(1.2); opacity: .9; }
   .cover { object-fit: contain; }
   ```

3. **核心技巧与框架无关**：`cover 铺底 + contain 前景 + blur + scale(1.2)` 这套组合在任何 CSS 环境（Vue / 原生 / Tailwind）都成立，只有语法外壳不同。
